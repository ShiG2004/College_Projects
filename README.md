# College Projects
Information system (Cybersecurity) projects at University of Pittsburgh at Johnstown. It has a collection of hands-on college projects covering cybersecurity, cloud computing, networking, system hardening, and IT infrastructure, with practical experience in AWS, Azure, Windows security, and network defense.

## Operating Systems & System Security
### Harden Windows OS:
Description: Applied operating system hardening techniques to reduce vulnerabilities in Windows. Configured security policies, disabled unnecessary services, and enforced secure configuration baselines. Verified improvements by reviewing system settings and security controls.<br>
**Tools: Local Security Policy, Windows Firewall Security, Oracle VirtualBox**

### Secure a Notebook:
Description: Implemented physical, firmware, and operating system security measures to protect a laptop from unauthorized access. Configured account security, encryption, and startup protections. This project focuses on endpoint security and real-world device protection practices.<br>
Tools: BIOS/UEFI, Command Prompt

### Hack a Notebook:
Description: Simulated common attack techniques against a notebook to identify system vulnerabilities. Documented weaknesses and demonstrated how attackers exploit misconfigurations. This project reinforces offensive security awareness and the importance of proactive defenses.<br>
Tools: Hiren's Boot CD, NTPWEdit, BIOS/UEFI, Windows OS ISO

### Recover Windows Password:
Description: Performed password recovery and reset procedures on a locked Windows system using authorized recovery techniques. Demonstrated understanding of Windows authentication mechanisms and system recovery processes.<br>
Tools: Hiren's Boot CD, NTPWEdit, Cleverfiles Disk Drill, Puran Software, Oracle VirtualBox, Server Manager

### Windows Log Files Analysis:
Description: Collected and analyzed Windows system and security logs to identify errors, suspicious activity, and authentication events. Used log data to understand system behavior and detect potential security incidents.<br>
Tools: Windows Event Viewer, Online IP Scanner

## Networking & Wireless Security
### Networking:
Description: Designed and analyzed basic network topologies, including IP addressing, routing, and connectivity. Explored how devices communicate across local and wide-area networks.<br>
Tools: Command Promt

### Subnetting Project:
Description: Created subnetting schemes to efficiently divide IP address spaces based on network size and requirements. Calculated subnet ranges, broadcast addresses, and usable hosts to optimize network design.<br>
Tools: Subnet Calculators, Manually Calculated

### Network Wireless Security:
Description: Configured wireless networks using secure authentication and encryption methods. Evaluated wireless security settings to reduce the risk of unauthorized access and attacks.<br>
Tools: Command Promt, WifilnfoView Application

### WiFi Vulnerability Scanning:
Description: Performed vulnerability scans on wireless networks to identify weak encryption, misconfigurations, and exposure to attacks. Documented findings and recommended remediation steps.<br>
Tools: Wireless Scanning Tools (Linksys), Fern WiFi Cracker, Kali Linux

### Spoofing & Poisoning:
Description: Demonstrated network spoofing and poisoning techniques to simulate man-in-the-middle attacks. Observed how traffic interception and redirection occur and discussed mitigation strategies.<br>
Tools: Technitium Mac Address Changer, Command Prompt

### Using Online Port Scanner:
Description: Scanned systems using online port scanning services to identify open ports and exposed services. Analyzed results to assess potential security risks and attack surfaces.<br>
Tools: Online Port Scanners (Hacker Target), Command Prommpt

## Cloud & Infrastructure Project
### AWS Firewalling Project:
Description: Designed and implemented firewall rules within an AWS environment to control network traffic between cloud resources and external networks. Configured inbound and outbound rules to allow only authorized ports, protocols, and IP addresses, then validated configurations by testing allowed and blocked traffic scenarios. This project demonstrates understanding of cloud network security, access control, and defense-in-depth principles.<br>
Tools: AWS, VPC, Security Groups, EC2

### AWS IAM RBAC:
Created and managed AWS IAM users, groups, and roles to implement role-based access control (RBAC). Applied least-privilege policies to restrict access to sensitive resources and tested permissions to ensure users could only perform authorized actions. This project emphasizes identity management, cloud security governance, and access auditing.<br>
Tools: AWS IAM, AWS Console

### Using Azure IaaS & PaaS:
Deployed virtual machines using Azure IaaS and implemented platform services using Azure PaaS. Compared deployment models by evaluating scalability, maintenance responsibility, and security implications. Demonstrated understanding of cloud service models and how organizations choose between infrastructure-level and platform-level solutions.<br>
Tools: Microsoft Azure, Azure VMs, Azure VNet

### Azure AD & DNS:
Description: Configured Azure Active Directory to manage user identities, authentication, and access control. Integrated DNS services to support domain name resolution within the Azure environment. This project demonstrates identity federation, directory services, and foundational cloud networking concepts.<br>
Tools: Azure AD, Azure DNS, Microsoft Azure, Azure VMs, Server Manager

### Modernize Datacenter:
Description: Analyzed an existing on-premises infrastructure and designed a modernization plan using virtualization and cloud technologies. Migrated workloads to a virtualized or cloud-based environment to improve scalability, reliability, and resource utilization. This project highlights infrastructure planning and cloud migration concepts.
Tool: Academic Technical Research
